# NurseDuty 개인정보처리방침 (Privacy Policy)

최종 수정: 2026-07-02

## 요약
NurseDuty는 **계정이 없고, 서버가 없으며, 모든 데이터는 사용자의 기기에만 저장됩니다.**
개발자는 사용자의 어떤 데이터도 수집·전송·열람하지 않습니다.

## 저장하는 데이터 (기기 내부에만)
- 근무표(날짜별 근무 배정), 듀티 프로필·알람·체크리스트, 빠른 메모, 사용자 이름(선택)
- 위 데이터는 앱 내부 데이터베이스(Room/SwiftData)에 저장되며, 사용자가 직접 실행하는
  JSON 내보내기 외에는 기기를 떠나지 않습니다.
- 페어링된 워치가 있으면 오늘의 근무 요약이 **기기 간 직접**(Wearable Data Layer)으로만 동기화됩니다.

## 네트워크 통신
앱이 외부와 통신하는 경우는 두 가지뿐이며, 어느 쪽도 개인 데이터를 보내지 않습니다.
1. **날씨**: 홈 화면 날씨 표시를 위해 open-meteo.com 에 현재 기온을 요청합니다(서울 고정 좌표,
   위치 권한을 사용하지 않음). 이때 통신 특성상 IP 주소가 open-meteo 서버에 전달됩니다.
2. **지참약 식별 탭**: yuangunn.github.io 의 정적 웹앱(ward-pillcheck)을 앱 안에서 불러옵니다.
   이 웹앱 역시 로컬 저장 방식이며 자세한 내용은 해당 프로젝트의 방침을 따릅니다.

## 권한
- **알림**: 근무 알람 표시용.
- **정확한 알람(SCHEDULE_EXACT_ALARM/USE_EXACT_ALARM)**: 간호사 출근·인계 알람이 앱의
  핵심 기능이므로 정시에 울리기 위해 사용합니다.

## 데이터 삭제
앱 삭제 시 모든 데이터가 함께 삭제됩니다. 앱 내 백업(JSON)을 만들었다면 해당 파일은
사용자가 저장한 위치에 남으므로 직접 삭제하면 됩니다.
